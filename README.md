# DESAFIO-QA-BEEDOO-2026


##Análise da aplicação
A aplicação tem como objetivo permitir o cadastro e a visualização de cursos em uma lista. O usuário pode inserir informações sobre um curso através de um formulário e posteriormente visualizar os cursos cadastrados.


##Fluxos do sistema

###Fluxo 1 — Cadastro de curso
  1. Usuário acessa a tela de cadastro
  2. Preenche os campos do formulário
  3. Clica em cadastrar curso
  4. Curso é adicionado na lista

###Fluxo 2 — Visualização de cursos
  1. Usuário acessa a lista
  2. Visualiza cursos cadastrados

###Fluxo 3 — Edição do curso
  1. Usuário acessa o curso
  2. Usuário edita dados do curso
  
###Fluxo 4 — Exclusão do curso
  1. Usuário acessa o curso
  2. Usuário exclui o curso


##Pontos críticos para teste
  - Validação de campos obrigatórios
  - Persistência dos dados após cadastro
  - Exibição correta na listagem
  - Tratamento de dados inválidos
  - Comportamento da aplicação ao enviar formulário vazio


##Explicação do raciocínio
Os cenários de teste foram elaborados com foco na verificação do funcionamento dos recursos apresentados, incluindo o fluxo principal de cadastro, a exibição dos cursos na listagem e alguns cenários negativos relacionados ao envio de dados incompletos ou inesperados. Como a aplicação não apresenta regras de negócio ou requisitos claramente definidos, aspectos como padrões específicos de formatação ou validações mais complexas não foram considerados em profundidade, priorizando-se a análise do comportamento geral da aplicação e a identificação de possíveis falhas no funcionamento das funcionalidades disponíveis.
